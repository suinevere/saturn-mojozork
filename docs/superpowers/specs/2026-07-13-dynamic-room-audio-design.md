# Dynamic Room Audio Atmospheres — Design

Saturn MojoZork port. Feature #15: pick and play looping CD-audio (Red Book)
music that follows the mood of the room the player is in, working on **any** v3
game out of the box, with optional per-game refinement.

Date: 2026-07-13. Branch context: `sound-lurking-horror`.

## 1. Goals / non-goals

**Goals**
- Universal: every v3 game gets mood music with **zero per-game authoring**.
- Sticky, non-jarring: music changes on room transitions and strong in-text
  events, not every turn; a same-mood room never restarts the track; a room that
  matches nothing keeps whatever is already playing (never abrupt silence).
- Optional per-game refinement without code changes (data-only override maps).
- Ship-safe today: with no audio tracks on the disc yet, the whole system
  no-ops silently and harmlessly.
- Coexist with the Lurking Horror PCM sound effects without the CD drive
  fighting itself.

**Non-goals (v1)**
- Authoring per-game room→category maps for the whole library (format ships;
  population is incremental, data-only, later).
- Music for v4+ games (room detection here relies on the v3 status-line global;
  v4+ falls back to no music, same as today).
- Crossfading / beat-matched transitions (CDDA is play/stop of whole tracks).

## 2. How "the current room" is known

`mojozork.c` already reads Z-machine **global variable 0** (`varAddress(0x10)`)
to render the v3 status line (`calculateStatusBar`): its value is the **current
location object number**, and `getObjectShortName` + `decode_zscii` yield the room
name. In v3 the object id is a single byte (objects ≤ 255). So "current room" is
a cheap, universal signal available every turn.

## 3. Categories (the semantic layer)

`NUM_CATEGORIES = 14`. Categories 1–11 are chosen by **room keywords**;
12–13 are **event-driven**; 0 is the neutral default / start state.

| # | Category | Source | Example rooms / games |
|---|----------|--------|-----------------------|
| 0 | Neutral / Exploration | default | anything unmatched; start state |
| 1 | Wilderness / Forest | room | Zork above-ground, gardens, fields |
| 2 | Underground / Cave | room | Zork caverns, tunnels, cellars |
| 3 | Water / Underwater | room | rivers, pools, Seastalker depths |
| 4 | Nautical / Ship | room | decks, cabins, Cutthroats, subs |
| 5 | Town / Domestic / Indoors | room | Deadline/Witness/Wishbringer houses |
| 6 | Dungeon / Ruins / Ancient | room | temples, tombs, Infidel, altars |
| 7 | Desert / Wasteland | room | Infidel sands, Starcross exterior |
| 8 | Magic / Mystical | room | Enchanter, Sorcerer, Spellbreaker |
| 9 | Sci-fi / Technological | room | Planetfall, Starcross, Stationfall |
| 10 | Horror / Dread | room | Lurking Horror, crypts |
| 11 | Mystery / Tension | room | detective interiors: Deadline, Witness, Suspect |
| 12 | Danger / Combat | event | monster/fight/fire moments |
| 13 | Triumph / Treasure | event | finding gold, winning |

Deliberate keyword overlaps exist (e.g. "ship" reads as both Nautical and
Sci-fi). The engine alone is best-effort; per-game override maps (§7) resolve
ambiguity where it matters.

## 4. Data tables (`music_data.c`)

All three are plain C data, hand-editable (optionally generated by a `tools/`
script later, mirroring `gen_solution.py`).

### 4.1 Category → track (many-to-one)

You have **fewer tracks than categories**, so several categories share a track.
Track 1 is the data track (the game); Red Book audio is tracks 2+. `0` means "no
dedicated track — keep whatever is playing".

```c
static const uint8_t CATEGORY_TRACK[NUM_CATEGORIES] = {
    /* 0 Neutral     */ 2,
    /* 1 Wilderness  */ 3,
    /* 2 Underground */ 4,
    /* 3 Water       */ 5,
    /* 4 Nautical    */ 5,   // shares Water
    /* 5 Town        */ 6,
    /* 6 Dungeon     */ 4,   // shares Underground
    /* 7 Desert      */ 3,   // shares Wilderness
    /* 8 Magic       */ 7,
    /* 9 Sci-fi      */ 8,
    /* 10 Horror     */ 9,
    /* 11 Mystery    */ 6,   // shares Town
    /* 12 Danger     */ 10,
    /* 13 Triumph    */ 11,
};
```

Edit this table to match the number of tracks actually recorded; collapse
categories onto shared tracks freely.

### 4.2 Room keyword table → categories 1–11

A flat list of `{category, word}` (whole-word, case-insensitive). Starter set
(non-exhaustive; finalized in implementation):

- Wilderness: forest, tree, trees, woods, grove, meadow, field, clearing, path, hill, mountain, garden, forest
- Underground: cave, cavern, tunnel, underground, cellar, mine, passage, grotto, crawlway
- Water: river, stream, lake, pool, water, waterfall, shore, bank, underwater, flooded
- Nautical: ship, boat, deck, cabin, hull, sea, ocean, dock, harbor, sail, mast, submarine
- Town/Domestic: house, hall, kitchen, office, street, building, stairs, parlor, room, bedroom
- Dungeon/Ruins: temple, tomb, crypt, ruin, altar, ancient, chamber, dungeon, catacomb, vault
- Desert: desert, sand, dune, oasis, wasteland, dry
- Magic: spell, magic, enchant, wizard, scroll, rune, mystic, tower, sorcerer
- Sci-fi: console, computer, airlock, panel, robot, laboratory, reactor, corridor, module, cockpit
- Horror: dark, blood, corpse, rotting, stench, shadow, eerie, decay, skeleton
- Mystery: body, clue, murder, evidence, study, library, safe, letter, detective, locked

`classify_text` counts hits per category over the turn's text and returns the
max (ties → lower category number; no hits → category 0 / caller keeps current).

### 4.3 Event word table → categories 12–13 (and bump 10)

Fired by any turn's text, override the room's base track until the next room
change:
- Danger (12): monster, troll, grue, attack, fight, flames, fire, burning, scream, danger
- Triumph (13): treasure, gold, jewel, chest, reward, gleaming, victory, won

## 5. Engine (`music.c/.h`)

Pure C, host-testable. Calls a `music_play_track(int track)` **callback** set at
init (like the `sound_effect` pointer), so it has no Saturn dependency in tests.

### 5.1 State
- `g_cur_room` (uint16) — last room seen.
- `g_base_cat` — the room's classified category.
- `g_event_cat` — active event override for the current room (−1 = none).
- `g_active_track` — track currently requested.
- `room_cat_cache[256]` (uint8) — `0` = unseen, else `category + 1`. ~256 bytes,
  O(1), instant on revisit (v3 objects ≤ 255).
- `g_turn_text[512]` — this turn's output, appended by `music_note_output`.

### 5.2 `music_on_turn(uint16 room)` — once per turn, at the top of the input-read hook
The core hook reads `room` from global 0 and calls this pure entry point (room is
a parameter so the state machine is host-testable without `GState`).
1. `room` is the current location object (from global 0).
2. Scan `g_turn_text` for event words → `event_cat` (or none).
3. If `room != g_cur_room` (entered a new room):
   - `base = per_game_map(room)` if a map matches the loaded game,
     else `room_cat_cache[room]` if cached,
     else `classify_text(g_turn_text)`; cache the result.
   - `g_cur_room = room`; `g_base_cat = base`; clear the event override.
4. If `event_cat` fired this turn, set `g_event_cat = event_cat`.
5. `target = (g_event_cat >= 0) ? g_event_cat : g_base_cat`.
6. `track = CATEGORY_TRACK[target]`. If `track != 0 && track != g_active_track`,
   `music_play_track(track)`; `g_active_track = track`. Else keep current.
7. Clear `g_turn_text`.

### 5.3 Placement
- `music_update()` is invoked at the start of the interpreter's input-read hook,
  after the room description has printed and before it blocks for the line.
- `saturn_writestr` (in `main.cxx`) calls `music_note_output(str, len)` in
  addition to `console_write`, so the engine sees raw game output.

## 6. Playback shim (`music_cdda.cxx`) and Options audio

### 6.1 CDDA shim
Implements `music_play_track` via `SRL::Sound::Cdda::PlaySingle(track, /*loop=*/true)`,
plus stop and a level setter. Only re-issues `PlaySingle` when the target track
differs from the playing one (the engine already guards this too).

**Open item:** confirm the SRL/SGL call for CD-DA **output level** (SCSP/CD
mixer). If only play/stop is exposed, the Music slider maps to a coarse level or
play/mute; resolved during implementation.

### 6.2 Options — two 7-level sliders (replace the Sound on/off row)
- **Music** (CDDA level) — default **7**.
- **PCM (Lurking Horror)** — default **4**.
- Range **0–7**, `0 = silence`, `7 = full`; Left/Right adjust. (If "7 levels"
  was meant strictly as 7 stops, trim to 0–6; the numbers 7/4 assume 0–7.)
- Music level → CDDA output level (0 stops CDDA). PCM level → existing
  sound-effect volume scale (0 = `sound_set_enabled(0)`).
- Persisted in the `MOJOOPTS` backup blob, replacing the single sound-flag byte
  with two level bytes; legacy blobs map old on/off → sensible defaults.

## 7. Per-game override maps

Keyed by story **release + serial** (Z header 0x02 / 0x12), like the typeahead
solution overlay: `{room_id → category}`. When the loaded game matches, its map
supersedes keyword classification for listed rooms. Ships empty (or with a couple
of examples) in v1; games are added later as **data only**, since word-detection
already covers every game.

## 8. Lurking Horror slice cache (CD contention fix, "A1")

`LRKHOROR.BLB` is ~570 KB — too large to preload whole (~350 KB HWRAM free after
a story loads). Instead:
- Refactor `sound.cxx` so loaded PCM slices live in a **persistent cache keyed by
  sound number**, owned for the game's lifetime rather than freed on stop.
- First trigger of each sound = one CD read (brief, one-time music dip); the loop
  and every later trigger play from RAM — no further CD contention with CDDA.
- Only the handful of slices LH actually triggers are cached, so it fits the
  budget.
- `sound.cxx` slots reference cached buffers instead of owning per-slot mallocs;
  `music_reset()` / game switch frees the cache.

## 9. Error handling / ship-safety
- No audio tracks on the disc today → `PlaySingle` no-ops; the system runs
  silently and harmlessly until tracks are added. Safe to ship immediately.
- Music level 0 → skip playback (or stop CDDA).
- Missing/failed CD read for a PCM slice → existing retry logic; cache stores
  only on success.
- Game switch / soft reset → `music_reset()`: stop CDDA, clear room cache, event
  override, slice cache, and `g_turn_text`.
- v4+ games → no valid v3 status-line global convention assumed; feature stays
  neutral (no music), unchanged from today.

## 10. Testing
- **Host unit tests** (like `test/sound_blorb_test.c`): drive `classify_text`
  with real sample room descriptions (Zork cavern, Planetfall airlock, Enchanter
  tower, Deadline study, Infidel tomb…) asserting expected categories; plus
  event-override and room-cache behavior. Pure C, no Saturn.
- **On-device (manual):** walk between rooms and confirm sticky, correct track
  changes; verify both sliders; confirm LH PCM effects and CDDA music coexist
  with at most a one-time dip per distinct sound.

## 11. Open questions / risks
1. **CD-DA output-level API** in SRL/SGL (see §6.1) — affects how granular the
   Music slider is. Coarse fallback acceptable.
2. **Slider range** — 0–7 (8 stops) assumed to match defaults 7/4; confirm vs a
   strict 7-stop reading.
3. **Track inventory** — `CATEGORY_TRACK` is a placeholder until the actual
   recorded tracks are known; collapsing categories is a data edit.
4. **Room description timing** — a few games print the room name via the status
   line only (not the body) on revisits; the room-cache handles revisits, and the
   first visit always prints a description, so this is expected to be fine.
